# Завдання 1. Сканер токенів OCaml

У цьому проєкті реалізовано простий  сканер для частини токенів мови OCaml.

Програма читає вхідний текст, проходить його посимвольно і формує список токенів.  Для кожного токена виводиться його тип, значення,, рядок, і колонка.

## Що розпізнає програма


Сканер розпізнає такі токени:

- `INT` — десяткові цілі чиисла, наприклад `34567`;
- `FLOAT` — числа з плаваючою комою, нп. `2.98`;
- `STRING` — звичайні рядкові літерали в подвійних лапках, нп.`"nano"`;
- `COMMENT` — коментарі  OCaml вигляду `(* comment *)`;
- `UNKNOWN` — це такі символи, які не підходять під жоден з описаних токенів.

Також додано обробку помилкових токенів:

- `ERROR_INT` — неправильний цілий літерал, наприклад `123abc`;
- `ERROR_FLOAT` — неправ. float, наприклад `.45` або `1.`;
- `ERROR_STRING` — рядок без закриваючих лапок;
- `ERROR_COMMENT` — коментар без закриття;
- `END_OF_FILE` — кінець вхідного тексту..

## Формат чисел

Цілі числа записуються тільки десятковими цифрами:

```text
123
0
4567
```

Числа з плаваючою комою мають формат:

```text
цифри.цифри
```

Наприклад:

```text
2.98
0.73
123.456
```

У цій реалізації такі записи як, наприклад, `.45` і `1.` вважаються помилковими float-літералами.

## Коментарі

Коментарі розпізнаюються у форматі OCaml:

```ocaml
(* this is a comment *)
```

Вкладені коментарі в цій версії нее обробляються.

## Рядки

Рядкові літерали записуються у подвійних лапках:

```text
"hello"
"abc cdef"
```

## Компіляція

```bash
g++ main.cpp  -o scanner.exe
```

## Запуск

Можна запустити програму і ввести текст вручну:

```bash
  ./scanner.exe
```

Також можна запустити з тестового файлу:

```bash
./scanner.exe < test_valid.ml
```

## Приклад вхідного файлу

```text
1234567
2.58
"abc cdef"
(* some example of comment *)
!
```

В результаті програмою будуть визначені такі токени:

```text
INT
FLOAT
STRING
COMMENT
UNKNOWN
END_OF_FILE
```

## Приклад файлу з помилками

```text
.45
1.
45dawb
"kgsrg
```

Результат :

```text
ERROR_FLOAT
ERROR_FLOAT
ERROR_INT
ERROR_STRING
END_OF_FILE
```

## Тестові файли

У проєкті є такі тестові файли:

- `test_valid.ml` — коректні токени;
- `test_invalid.ml` — приклад некоректного введенняз помилками у float, int і string;
- `test_invalid_comment.ml` — приклад незакритого коментаря.

Приклади запуску:

```bash
./scanner.exe < test_valid.ml
./scanner.exe < test_invalid.ml
./scanner.exe < test_invalid_comment.ml
```

## Коротке пояснення реалізації

У коді є клас `Scanner`, який зберігає вхідний текст, поточну позицію, номер рядка і колонки.

Основні функції:

- `currentChar()` — повертає   поточний символ;
-  `peek()` — дивиться наступний символ;
- `advance()` —   переходить до наст. символу;
- `addToken()` — додає токен у список;
-  `scanNumber()`   — обробляє числа;
- `scanString()` — обробляє рядки;
- `scanComment( )` — обробляє коментарі;
- `scanUnknown()` — обробляє невідомі символи.

Головний цикл у функції `scan( )` проходить весь текст до кінця. Якщо символ підходить під якийсь токен, тоді викликається відповідна функція. Якщо ні, додається токен `UNKNOWN`.

Таким чином програма обробляє весь вхідний текст і формує послідовність токенів з їх типами.
